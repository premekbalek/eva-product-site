Proč EVA

EVA není jen AI. Je to orchestrační vrstva, díky které zákaznická péče skutečně funguje.

Firmy už mají data, API a systémy. Často jim ale chybí způsob, jak je proměnit ve funkční a škálovatelný systém zákaznické interakce.

Data a API sama o sobě zákaznickou péči nevytvoří. Tu vytváří orchestrace.

Co si reálně kupujete

Nekupujete:

  • chatbota
  • LLM model
  • integrační vrstvu

Kupujete:

Schopnost řešit zákaznické požadavky end-to-end bez operátora.

Multi-agent orchestrace

EVA je koordinovaný agentní systém:

Master agent
  • rozumí záměru
  • řídí celé flow
  • rozhoduje o dalším kroku
Specializovaní agenti
  • řeší konkrétní use cases (fakturace, objednávky, support)
  • vykonávají business logiku
Backendové systémy
  • poskytují data
  • neřídí samotné flow

Logika žije v agentní vrstvě, ne v backendových systémech.

Data jako rozhodovací vrstva

Tradiční systémy:
→ vrací data

EVA:
→ používá data pro rozhodování

  • určí, kdy volat API
  • kombinuje více datových zdrojů
  • pracuje i s neúplnými daty

Nevracíme jen data. Jednáme na jejich základě.

Integrace na vyžádání

  • Bez těžké integrace na začátku
  • Data se volají jen tehdy, když jsou potřeba

Dopad:

  • rychlejší nasazení
  • nižší latence
  • nižší náklady

Proč je těžké postavit to interně

Organizace umí postavit:

  • APIs
  • chatbots
  • AI modely

Ale naráží u:

  • koordinace agentů
  • návrhu interakčních flow
  • práce s neúplnými daty
  • řízení nákladů na LLM
  • opakovaného použití logiky napříč use cases

Problém není AI. Problém je orchestrace.

Co EVA přináší

  • Ověřený multi-agent orchestrační model
  • Reálné produkční zkušenosti
  • Znovupoužitelné interakční patterny
  • Optimalizovaný nákladový model (LLM + pravidla + API)
  • Oddělení logiky od backendových systémů

EVA mění systémy a data ve funkční zákaznickou péči.

Bez orchestrace zůstává AI demem. EVA ji posouvá do produkce.

EVA může využívat stejný platformní základ jako Virtual Buddy, ale míří ven na zákaznické journeys, backendový kontext a řízené vykonávání akcí.

EVA Perform&Operate

Plně spravováno O2

  • Provoz a monitoring
  • Průběžné ladění
  • Minimální provozní zátěž zákazníka

Volitelné doplňky

  • SARA: Kontrola kvality odpovědí a souladu s pravidly
  • Widget: Vložený web chat s rychlým nasazením
  • AI Voice: Přirozené hlasové interakce
  • Aplikace: Dedikovaný mobilní nebo webový support kanál

Infrastrukturní specifika

  • Data/action plane: Zákaznický kontext, oprávnění a backend action endpointy musí být řízené a auditovatelné.
  • Runtime riziko: EVA může provádět zákaznické akce, takže identita, autorizace, rollback a handoff na operátora jsou povinná témata návrhu.
  • Preferovaný enterprise model: Hybridní control plane s customer data/action plane pro citlivé nebo transakční use cases.

Virtual Buddy

Interní znalostní AI

Pro zaměstnance, kteří potřebují rychlý a přesný přístup k firemním znalostem a procesní logice.

Vstupní kanály

Web Chat Internal UI API (Custom varianta)

Platformní základ

Virtual Buddy je interní znalostní varianta společné platformy: dokumenty, konektory, RAG, testing, policy management a LLM governance.

Servisní varianty

VB Start

Okamžitý přístup ke znalostem

  • AI chat nad interní znalostní bází
  • Podpora zdrojů Confluence a SharePoint
  • Nízká komplexita nasazení
VB Boost

Spolehlivé a konzistentní odpovědi

  • Vše z VB Start
  • Čištění a strukturování znalostí
  • Ladění odpovědí a AI testování
VB Custom

Enterprise správa znalostí

  • Vlastní logika a workflow automatizace
  • Systémové integrace
  • Pokročilý monitoring a governance

Volitelné doplňky

  • SARA: Validace kvality a relevance interních odpovědí
  • Widget: Vložený asistent v zaměstnaneckých nástrojích
  • AI Voice: Hands-free přístup ke znalostem
  • Aplikace: Mobilní nebo webová aplikace pro interní týmy

Infrastrukturní specifika

  • Knowledge plane: FE, document storage, query/audit databáze, vector index a zdrojové konektory tvoří jádro řešení.
  • LLM vrstva: Lite LLM s PTU předplatným by měl jít přes O2 gateway, s pay-as-you-go fallback modely pro špičky.
  • Preferovaný start model: O2-hosted SaaS pro rychlý start, hybrid private connect pro enterprise požadavky na kontrolu dat.
